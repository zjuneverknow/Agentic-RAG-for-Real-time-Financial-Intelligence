# Skill: 显式推理模块集成 (Reasoning Integration)

在 RAG 2.0 架构中，推理模块负责在“检索”与“生成”之间建立逻辑桥梁，解决传统 RAG 缺乏证据关联建模的问题。

## 1. 推理范式路由表 (Paradigm Selection)
根据金融业务的逻辑复杂度和计算资源，选择最适配的推理路径：

| 推理范式 | 核心技术实现 | 适用金融场景 | 复杂度 |
| :--- | :--- | :--- | :--- |
| **精炼型 (RAG+)** | 任务感知推理网络 + 证据重排序/过滤 | 季度财报分析、公告关键点提取 | 低 |
| **模拟型 (CoT-RAG)** | 知识图驱动思维链 + 伪程序化提示 | 可转债定价、信用利差预测 | 中 |
| **多跳型 (HopRAG)** | 迭代子查询生成 + 文档节点路径遍历 | 跨国并购追踪、股权关系溯源 | 高 |
| **优化型 (ReARTeR)** | PRM/PEM 奖励模型 + 蒙特卡洛树搜索 | 期权对冲策略、复杂资产配置 | 极高 |
| **图构建型 (TRACE)** | 三元组提取 + 自回归链路构造 | 基金业绩归因、关联风险识别 | 高 |
| **分解验证 (RARE)** | 子问题拆解 + RAFS 事实性评分器 | 反洗钱合规审查、尽职调查 | 中 |



## 2. 核心集成流程 (Integration Workflow)

### A. 概念识别与过滤
*   **输入：** 原始检索文档列表 + 用户原始需求。
*   **动作：** 提取核心金融概念（如“ROE”、“现金流”、“违约风险”），基于预设规则过滤噪声段落。

### B. 逻辑链构建
*   **思维链触发：** 引导 LLM 生成中间思考步骤，确保推理过程透明。
*   **图谱辅助：** 若涉及实体关联，调用知识图谱接口，在离散证据间建立逻辑边。

### C. 证据优先级排序
*   **异常检测：** 识别数据中的同比/环比异常波动，将其关联证据标记为“高优先级”。
*   **权重打分：** 根据任务相关性模型，对候选推理链路进行评分。

## 3. 组件化封装要求 (Module Encapsulation)

为了保证系统的可扩展性与可审计性，推理模块需满足以下工程要求：

*   **标准接口定义：**
    *   `Input`: 接收标准化 JSON 格式的检索结果。
    *   `Output`: 输出带有置信度评分的结构化证据链。
*   **决策透明化 (Traceability)：**
    *   系统必须设计详细的日志格式，记录每一步推理的**输入**、**决策理由**以及**所引用的原始文档索引**。
*   **可插拔设计：**
    *   支持根据配置文件动态切换推理范式（例如：简单查询走 RAG+，复杂查询自动切换至 HopRAG）。

## 4. 关键产出 (Key Deliverable)
*   **精炼证据列表：** 经过逻辑排序、去噪后的上下文信息。
*   **中间推理轨迹：** 为最终生成层提供逻辑支撑，防止模型产生“幻觉”或“逻辑断层”。
